myrepos setup-script generator

If you use myrepos, you know it's great: with one command myrepos updates all your local repos. But, if you have a lot of repos, registering them with mr register one by one is a pain. And if you back up all your repos somewhere, you can forget which repos you've added the backup. To set up a thorough local backup, I needed a script that clones all of my repos and registers them with myrepos in one go.

This page generates the content for a shell script or batch file that I use to clone all of my GitHub repos and register them with myrepos. It may work for you, too.

Note: this only works for up to 100 repos.

  1. To get your organisation's repos, enter its GitHub name below. To get your personal repos, enter your GitHub username.
  2. If you want to include private repos, you'll need a GitHub Personal API token with full repo scope.
  3. Click 'Generate'.
  4. To add another organisation's or user's repos, add their details to the form and click 'Generate' again.
  5. Copy and paste the text into a .sh or .bat file.
  6. Run that file where you want your repos. They will be cloned there in organisation folders.
