myrepos setup-script generator

If you use myrepos, you know it's great: once you've cloned your repos and registered them, with one command myrepos updates all your local repos.

But, if you're using it to back up all your repos somewhere, and you have a lot of them, cloning and registering them one by one is a pain. And you can forget which repos you've cloned or registered.

To maintain a thorough local backup, I needed a shell script that would clone all of my repos and register them with myrepos in one go. And I needed to be able to get an up-to-date version of that script, that included any new repos, at any time.

This page generates that script. It may work for you, too.

Note: currently, this only works for GitHub repos and up to 100 repos per user or organisation.

Usage

  1. Install myrepos.
  2. To get your organisation's repos, enter its GitHub name below. To get your personal repos, enter your GitHub username.
  3. If you want to include private repos, you'll need a GitHub Personal API token with full repo scope.
  4. Click 'Generate'.
  5. To add another organisation's or user's repos, add their details to the form and click 'Generate' again.
  6. Copy and paste the text into a .sh or .bat file.
  7. Run that file where you want your repos. They will be cloned there in organisation folders. You do need to be set up to connect to GitHub with ssh automatically.
  8. Once that's done, to update all your repos with myrepos, run mr update in your home folder.
